# The Film Atlas

The Film Atlas is a standalone portfolio project for exploring 10,000 films by
experiential similarity: story, mood, texture, theme, and audience perception.
It combines an offline Python data pipeline with a static Astro frontend that
renders the finished atlas in the browser.

The public app is intentionally static. The expensive and private work happens
offline: data collection, profile construction, embeddings, clustering, label
generation, validation, and export. The browser receives only sanitized JSON and
draws the atlas with a custom Canvas renderer.

## Repository Structure

- `film_atlas/`: Python package and CLI for the offline pipeline.
- `tests/`: fixture-based Python tests.
- `docs/`: project briefs, milestone reports, implementation notes, and audit
  writeups.
- `outputs/public_export/`: sealed static export generated by the pipeline.
- `frontend/`: standalone Astro frontend for viewing and using the atlas.
- `frontend/public/film-atlas/data/`: copy of the sealed export served by Astro.

## Run The Frontend

Install frontend dependencies:

```bash
cd frontend
pnpm install
```

Start the local Astro server:

```bash
pnpm dev
```

Open the app at:

```text
http://localhost:4321/
```

The same experience is also available at:

```text
http://localhost:4321/film-atlas/
```

Build the static site:

```bash
pnpm build
```

Preview the built site:

```bash
pnpm preview
```

## Public Export

The frontend reads from `frontend/public/film-atlas/data/`. That directory is a
served copy of `outputs/public_export/`.

The export contains:

- `manifest.json`
- `movies.json`
- `points.json`
- `macro_clusters.json`
- `neighborhood_clusters.json`
- `micro_clusters.json`
- `labels.json`
- `territory_layouts.json`
- `neighbor_shards/*.json`

The public export should not contain API keys, raw reviews, raw embeddings, or
private experiment artifacts.

After regenerating `outputs/public_export/`, refresh the frontend copy:

```bash
rsync -a --delete outputs/public_export/ frontend/public/film-atlas/data/
```

## Pipeline Setup

Install Python dependencies with `uv`:

```bash
uv sync
```

If `uv` is installed but not on your shell path, add the directory that contains
it:

```bash
export PATH="$HOME/.local/bin:$PATH"
```

Copy the environment template:

```bash
cp .env.example .env
```

Add credentials to `.env` only for live pipeline runs:

```bash
TMDB_BEARER_TOKEN=your_tmdb_read_access_token
OPENAI_API_KEY=your_openai_api_key
```

Do not commit `.env`, API keys, raw caches, raw reviews, embeddings, or private
intermediate outputs.

Optional local path controls:

```bash
FILM_ATLAS_DATA_DIR=data
FILM_ATLAS_OUTPUT_DIR=outputs
```

## Common Pipeline Commands

Small local proof run:

```bash
uv run film-atlas quickstart --limit 100
```

Balanced TMDb sampling:

```bash
uv run film-atlas fetch-balanced --per-decade 100 --start-year 1980 --end-year 2026
uv run film-atlas fetch-details
uv run film-atlas normalize
uv run film-atlas build-profiles --review-weight light --max-review-chars 180
```

Embedding and clustering steps:

```bash
uv run film-atlas estimate-embeddings --limit 100
uv run film-atlas embed-profiles --limit 100
uv run film-atlas reduce-embeddings
uv run film-atlas cluster-movies
uv run film-atlas compute-neighbors
uv run film-atlas inspect-clusters
```

Scaled atlas export:

```bash
uv run film-atlas milestone-4 --target 2000
```

Or run the export-oriented pieces separately:

```bash
uv run film-atlas scale-dataset --target 2000 --since-year 1980 --min-votes 100
uv run film-atlas embed-profiles
uv run film-atlas build-hierarchy --macro-k 12 --neighborhood-k 75 --micro-k 200
uv run film-atlas export-atlas-data
```

## Verification

Python tests and linting:

```bash
uv run pytest
uv run ruff check .
```

Frontend build:

```bash
cd frontend
pnpm build
```

Export privacy scan:

```bash
rg -i "api[_-]?key|bearer|embedding|embeddings|raw_review|review_text" outputs/public_export frontend/public/film-atlas/data
```

The privacy scan should not return secret-bearing public data. If it finds a
field name, inspect it before publishing.

## Data Sources

The atlas uses controlled public signals for a non-commercial portfolio project:

- TMDb for movie identity and metadata.
- MovieLens Tag Genome for structured audience-perception signal.
- MPST for plot tags and controlled synopsis texture.
- Light, capped review-language snippets during offline profile construction.

The static frontend does not call TMDb, OpenAI, or any recommendation API at
runtime.
